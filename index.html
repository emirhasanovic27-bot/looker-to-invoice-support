▸ Conversion rules
DESCRIPTION
Hourly rows become Invoice for MM/DD/YYYY - MM/DD/YYYY using the date range in the source; stray spacing collapsed. Bonus & Refund rows keep their original (cleaned) text.
TEAM/LANGUAGE
Contains "PM" → PM; contains "Leads" → Leads; otherwise language from the G-code prefix — every en* code → English, all others per prefix (ja→Japanese, it→Italian, es-419→Spanish LatAm, …).
G-CODE backfill
If a row has no G-code, the converter first checks whether the username has a single unique G-code elsewhere and uses it. Otherwise it derives the rate as Total Billing ÷ Hours and matches it to that user's known bill rates — recovering the exact G-code and rate (e.g. 72.60 ÷ 10 = 7.26 → that user's en-1-s row). The matched code and rate are filled in. Rows that still can't resolve get flagged.
PROGRAM MAPPING
Content Curation → Content Cur, Multimodal, Schema · 1P feedback → 1P Fdbk · AIA feedback → AIA Fdbk.
HOURS
40:00 → 40 · 37:10 → 37.17 (decimal). Negative source hours are made positive. Bonus/Refund rows carry no hours.
USERNAME
Always written as plain text — numeric-looking handles like 026779 keep their leading zeros and never become numbers.
AMOUNTS
Currency text parsed to numbers; refund negatives preserved.
